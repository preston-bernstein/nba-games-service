# Commit Message Conventions

Use a concise, present-tense summary. Preferred format:

- `feat: ...` for new features
- `fix: ...` for bug fixes
- `chore: ...` for maintenance (deps, CI, docs, tooling)
- `refactor: ...` for non-behavioral code changes
- `docs: ...` for documentation-only changes

Examples:
- `feat: add readiness probe`
- `fix: validate date query for games endpoint`
- `chore: add CI workflow and Dockerfile`
